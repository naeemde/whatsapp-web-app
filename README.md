# واتساب ويب - تطبيق أندرويد

تطبيق أندرويد بسيط يفتح **WhatsApp Web** (نفس النسخة التي تفتحها على متصفح اللابتوب:
web.whatsapp.com) داخل شاشة تطبيق مستقلة بدل المتصفح العادي. تربط حسابك عبر مسح رمز QR
تمامًا مثل ما تفعل على الكمبيوتر، وبعدها تكتب وترسل صور وملفات وتستقبل كل شيء.

- **الحد الأدنى المدعوم:** أندرويد 5.0 Lollipop (API 21) فما فوق — راجع قسم
  "ملاحظة عن الأجهزة القديمة جدًا" بالأسفل.
- **compileSdk/targetSdk:** 35.

---

## الطريقة 1: بناء الـ APK تلقائيًا عبر GitHub (بدون تثبيت أي برنامج)

المشروع فيه ملف جاهز `.github/workflows/build.yml` يخلي GitHub نفسه يبني لك ملف
APK جاهز للتثبيت، تحمّله مباشرة من الموقع.

1. سجّل دخول على github.com وأنشئ **مستودع (Repository) جديد فاضي** — لا تضف
   README أو .gitignore تلقائيًا حتى لا يتعارض مع ملفات المشروع.
2. فك ضغط الأرشيف عندك، وارفع **محتويات** مجلد `WhatsAppWebApp` (وليس المجلد نفسه)
   إلى جذر المستودع مباشرة. أسهل طريقة بدون سطر أوامر:
   - من صفحة المستودع الفارغة اضغط "uploading an existing file"
   - اسحب كل الملفات والمجلدات (`app`, `gradle`, `.github`, `build.gradle`, إلخ) دفعة وحدة
   - تأكد إن `settings.gradle` ينتهي به المطاف في **جذر** المستودع، مو داخل مجلد فرعي
3. بعد الرفع، افتح تبويب **Actions** أعلى صفحة المستودع. سيبدأ بناء تلقائي
   (أو اضغط "Run workflow" يدويًا لو ما بدأ).
4. انتظر حتى تخلص العملية (دقيقتين إلى خمس تقريبًا) بعلامة ✅ خضراء.
5. ادخل على التشغيلة (Run) اللي خلصت، وبالأسفل بقسم **Artifacts** بتلقى ملف
   اسمه `whatsapp-web-debug-apk` — حمّله (ينزل كملف zip يحتوي app-debug.apk بداخله).
6. انقل ملف الـ APK لجوالك، وثبّته (فعّل "السماح بالتثبيت من مصادر غير معروفة"
   لو طلب منك أندرويد ذلك أول مرة).

> ملاحظة: هذا APK "تجريبي/Debug" غير موقّع بشهادة نشر رسمية — يعمل تمامًا على جهازك
> الشخصي، لكن لا يصلح للرفع على متجر Google Play كما هو (يحتاج توقيع Release منفصل).

## الطريقة 2: عبر Android Studio على جهازك

1. Android Studio ← New Project ← **Empty Views Activity** ← Language: **Kotlin**.
2. Package name: `com.example.whatsappweb` — Minimum SDK: **API 21**.
3. انسخ محتوى `MainActivity.kt`, `activity_main.xml`, `AndroidManifest.xml`, وملف
   الأيقونة من هذا الأرشيف لنفس المسارات بمشروعك الجديد.
4. أضف بملف `build.gradle` (أو `build.gradle.kts`) الخاص بمجلد `app`:
   - Groovy: `implementation 'androidx.swiperefreshlayout:swiperefreshlayout:1.1.0'`
   - Kotlin DSL: `implementation("androidx.swiperefreshlayout:swiperefreshlayout:1.1.0")`
5. Sync ثم Run ▶️ (على جوال حقيقي موصول بالـ USB أو محاكي Emulator).

أو جرّب فتح مجلد المشروع المرفق مباشرة (File → Open)، ووافق إذا طلب منك أندرويد
ستوديو ترقية إصدار Gradle/AGP عند الفتح.

---

## طريقة الاستخدام
1. شغّل التطبيق.
2. يظهر رمز QR بنفس شكله على الكمبيوتر.
3. من هاتفك اللي عليه واتساب: الإعدادات ← الأجهزة المرتبطة ← ربط جهاز ← امسح الرمز.
4. تصفّح محادثاتك، أرسل واستقبل صور وملفات بشكل طبيعي.

## ⚠️ ملاحظة عن الأجهزة القديمة جدًا
خفّضت الحد الأدنى إلى Android 5.0 (API 21) لأنه أول إصدار صار فيه مكوّن WebView
(محرك عرض الصفحات داخل التطبيق) قابل للتحديث بشكل منفصل عن نظام التشغيل عبر متجر
Google Play — يعني حتى لو الجهاز قديم، المتصفح الداخلي نفسه ممكن يكون محدّثًا.

**لكن** المشكلة الحقيقية مو رقم إصدار أندرويد، بل نسخة WebView الفعلية المثبتة
على الجهاز:
- إذا الجهاز أقدم من Android 5 (Lollipop)، أو معطّل عنده تحديث خدمات جوجل/المتجر،
  فمحرك WebView يكون قديم جدًا وقد لا يقدر يعرض موقع واتساب ويب أصلًا (صفحة فاضية
  أو تصميم مكسور) — بغض النظر عن كود التطبيق نفسه.
- الحل الوحيد لهذي الحالة هو تحديث "Android System WebView" من متجر Google Play
  على ذلك الجهاز (إن كان المتجر شغّال عنده).
- بمعنى آخر: التطبيق نفسه بيثبّت ويفتح على أجهزة قديمة جدًا، لكن تجربة الاستخدام
  الفعلية تعتمد على مدى حداثة WebView المثبت، مو على كود التطبيق.

## حدود إضافية
- المكالمات الصوتية/المرئية عبر واتساب ويب قد لا تعمل بشكل كامل داخل WebView.
- هذا "غلاف" (wrapper) لموقع web.whatsapp.com الرسمي نفسه، وليس تطبيقًا يتواصل مع
  واتساب بطريقة غير رسمية.
- يحتاج جهازك يبقى متصل بالإنترنت من وقت لآخر، تمامًا كواتساب ويب العادي.
